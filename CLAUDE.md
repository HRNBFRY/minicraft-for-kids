# 開発ルール

このプロジェクト（マイクラ風ゲーム）を編集する際は、以下のルールを必ず守ること。

## 最優先ルール

**1. 作業開始前に、必ず最新のリモート状態を取得する。**

- git pull を実行し、コンフリクト（分岐を含む）がある場合は作業を中断してユーザーに報告する。
- pull を行わずに編集を開始してはいけない。

**2. セーブデータ（ワールドデータ）はユーザーが遊んで作成した大切なデータ。**

- GitHub リポジトリ側にある worlds/・profiles/ 配下のセーブデータを削除・初期化・上書きしてはいけない。
- セーブデータの保存形式を変更する場合は、既存データとの互換性を維持する。
- 必要がない限り、セーブデータ関連ファイルには一切変更を加えない。

**3. コードの修正対象は、ゲーム本体（src/js、assets/css、index.html、UI 等）のみ。**

- worlds/・profiles/ のセーブデータファイル自体はコード修正の対象外とする。

**4. 作業終了後は変更内容を確認し、問題がなければコミットする。 コミット前に必ず以下を確認する。**

- セーブデータが変更されていないこと
- 不要なファイル（.DS_Store 等）が含まれていないこと
- ビルドエラーがないこと

**5. コミット後は git push を実行する。 push が拒否された場合は、勝手に force push せず状況を報告する。**

## 禁止事項

- git push --force
- git reset --hard
- セーブデータの削除・初期化
- ユーザーへの確認なく履歴を書き換える操作

## 不明な場合

データ消失の可能性が少しでもある場合は、変更を行わずユーザーに確認する。
